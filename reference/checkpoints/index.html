    CheckpointMetadata ¶

    基类: TypedDict

    与检查点关联的元数据。

    属性

    名称 类型 描述

    source instance-attribute ¶

    source: Literal['input', 'loop', 'update', 'fork']

    检查点的来源。

    • "input": 检查点是从invoke/stream/batch的输入创建的。
    • "loop": 检查点是在pregel循环内部创建的。
    • "update": 检查点是从手动状态更新创建的。
    • "fork": 检查点是作为另一个检查点的副本创建的。

    step instance-attribute ¶

    step: int

    检查点的步骤编号。

    对于第一个“input”检查点为-1。对于第一个“loop”检查点为0。...对于其后的第n个检查点。

    writes instance-attribute ¶

    writes: dict[str, Any]

    在前一个检查点和当前检查点之间进行的写入。

    从节点名称到该节点发出的写入的映射。

    parents instance-attribute ¶

    parents: dict[str, str]

    父检查点的ID。

    从检查点命名空间到检查点ID的映射。

    Checkpoint ¶

    基类: TypedDict

    某个时间点的状态快照。

    属性

    名称 类型 描述
    v int

    检查点格式的版本。目前为1。
    id str

    检查点的ID。此ID既唯一又单调
    ts str

    检查点的时间戳，ISO 8601格式。
    channel_values dict[str, Any]

    检查点时通道的值。
    channel_versions ChannelVersions

    检查点时通道的版本。
    versions_seen dict[str, ChannelVersions]

    从节点ID到通道名称到已见版本的映射。
    pending_sends List[SendProtocol]

    已推送到节点但尚未处理的输入列表。

    v instance-attribute ¶

    v: int

    检查点格式的版本。目前为1。

    id instance-attribute ¶

    id: str

    检查点的ID。此ID既唯一又单调递增，因此可用于从第一个到最后一个检查点的排序。

    ts instance-attribute ¶

    ts: str

    检查点的时间戳，ISO 8601格式。

    channel_values instance-attribute ¶

    channel_values: dict[str, Any]

    检查点时通道的值。从通道名称到反序列化通道快照值的映射。

    channel_versions instance-attribute ¶

    channel_versions: ChannelVersions

    检查点时通道的版本。键是通道名称，值是每个通道单调递增的版本字符串。

    versions_seen instance-attribute ¶

    versions_seen: dict[str, ChannelVersions]

    从节点ID到通道名称到已见版本的映射。这跟踪每个节点已见的通道版本。用于确定接下来要执行的节点。

    pending_sends instance-attribute ¶

    pending_sends: List[SendProtocol]

    已推送到节点但尚未处理的输入列表。在下一个检查点时清除。

    SerializerProtocol ¶

    基类: UntypedSerializerProtocol, Protocol

    对象序列化和反序列化的协议。

    • dumps: 将对象序列化为字节。
    • dumps_typed: 将对象序列化为元组 (type, bytes)。
    • loads: 从字节反序列化对象。
    • loads_typed: 从元组 (type, bytes) 反序列化对象。

    有效的实现包括pickle、json和orjson模块。

    如果您正在使用LangGraph平台，则无需指定检查点器。将自动使用正确的托管检查点器。

        import asyncio

    from langgraph.checkpoint.memory import InMemorySaver

    具有与shelve和anydbm兼容的API的持久化字典。

    字典保存在内存中，因此字典操作的运行速度与普通字典一样快。

    AsyncSqliteSaver ¶

    基类: BaseCheckpointSaver[str]

    一个将检查点存储在SQLite数据库中的异步检查点保存器。

    此类别提供了一个异步接口，用于使用SQLite数据库保存和检索检查点。它专为异步环境设计，并为I/O密集型操作提供比同步替代方案更好的性能。

    属性

    名称 类型 描述
    conn Connection

    异步SQLite数据库连接。
    serde SerializerProtocol

    用于编码/解码检查点的序列化器。
    提示

    需要aiosqlite包。使用pip install aiosqlite安装。

    警告

    尽管此类别支持异步检查点，但由于SQLite的写入性能限制，不建议用于生产工作负载。对于生产用途，请考虑使用更强大的数据库，例如PostgreSQL。

    提示

    请记住在执行代码后关闭数据库连接，否则，您可能会看到图形在执行后“挂起”（因为程序在连接关闭之前不会退出）。

    最简单的方法是使用示例中所示的async with语句。

    async with AsyncSqliteSaver.from_conn_string("checkpoints.sqlite") as saver:
    # Your code here
    graph = builder.compile(checkpointer=saver)
    config = {"configurable": {"thread_id": "thread-1"}}
    async for event in graph.astream_events(..., config, version="v1"):
        print(event)
